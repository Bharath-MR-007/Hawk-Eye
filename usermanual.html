User Manual

A complete guide for day-to-day operation of the Hawk-Eye monitoring dashboard. No prior monitoring experience required.

Getting Started

After a successful installation, open your browser and navigate to http://localhost:8080. You will be greeted by the login page.

  1. Log InEnter your credentials. The default is admin / admin. Change your password immediately after your first login via the Admin settings.
  2. You're on the Live DashboardThis is the home screen. It shows real-time health and latency data for all configured targets. A green dot means a target is UP, red means it's DOWN.
  3. Configure your first targetIf no targets are shown, go to the Inventory page (top nav) to add targets, or ask your administrator to update checks.yaml.

Live Dashboard

The Live Dashboard provides a WebSocket-based real-time view of your entire monitored infrastructure. Data refreshes automatically without needing to reload the page.

Reading the Status Table

Each row represents a monitored target. The columns cover all monitoring layers simultaneously:

Column Check Type Description
Application / Target — The URL or hostname being monitored, with favicon icon
Tags — User-defined labels (e.g., Production, TSI, Shopping). Click to edit.
L7 Health (HTTP) health 🟢 Healthy / 🔴 Down — based on HTTP response code and retry logic
Latency latency Round-trip time in milliseconds. Shows first-byte time as sub-text.
DNS Resolution dns ✅ Resolved / ❌ Failed — DNS lookup status and server used
TCP Handshake tcp_metrics Time (ms) to establish a raw TCP connection to the target port
SSL/TLS Validity ssl_tls Days remaining until the TLS certificate expires. Shows warning if < 30 days.
Network Path traceroute Minimum hop count to reach the target (e.g., "0 Hops — Restricted by Docker")
Packet Loss traceroute Percentage of hops that did not respond. 0.0% = clean path.

Using the Search Bar

Type in the search bar (top of the page) to filter targets in real time. The filter applies to target names, IPs, and tags simultaneously. Press Esc to clear.

Using the Incident Summary

If any targets are DOWN, the dashboard will show an "Incident Summary" banner at the top with a count and links to open Grafana or Alertmanager for deeper investigation.

Inventory Page

The Inventory page is accessible from the top navigation bar. It provides a full list of all configured monitoring targets with their associated metadata.

Grouping Targets

Use the Group By dropdown (top right of the inventory) to organize targets:

  • None — a flat alphabetical list of all targets
  • Tags — groups targets by their assigned tags (e.g., all production targets together)
  • Name (Regex) — use a regular expression to group targets by naming pattern (e.g., ^api- groups all targets that start with "api-")

Alert Configuration

The Alerts → Config page allows you to view and edit Prometheus alerting rules that control when notifications are sent.

  • Select a target from the left sidebar to view its current alert rules
  • Click Edit to modify thresholds (e.g., alert if latency > 500ms for 5 minutes)
  • Changes are applied immediately via the Prometheus /-/reload API

Incorrect YAML syntax in alert rules will cause the Prometheus reload to fail. Always validate your YAML before saving.

Tagging & Grouping

Tags are stored in your browser's localStorage and can be applied or removed from the dashboard sidebar. They allow you to create logical groupings:

  • Tags like production, eu-west, database let you filter the dashboard to see only relevant targets
  • Multiple tags can be active simultaneously for complex filter queries
  • Tags are exported as part of the full configuration backup

Backup & Restore (Export / Import)

Hawk-Eye supports a one-click backup of the entire configuration, which can be restored on a new instance. This is useful for migrations, DR testing, or version-controlled config management.

Export Configuration

  1. Click "Export" in the top navigation barA progress notification will appear at the bottom right of the screen.
  2. A ZIP file will be downloadedNamed hawk-eye-full-config-YYYY-MM-DD.zip. Keep this file secure — it contains your alert thresholds and notification configurations.

Import Configuration

  1. Click "Import" in the top navigation barA confirmation dialog will warn you that existing configuration will be overwritten.
  2. Select your ZIP backup fileThe system will unpack the archive and load all configuration files.
  3. Services reload automaticallyPrometheus and Alertmanager will reload their configs within a few seconds. The dashboard will refresh.

Import will overwrite all current configuration files. Ensure you have an existing backup before importing an older one.
