Software Requirements

Minimum and recommended system specifications for running Hawk-Eye in production environments.

Hardware Requirements

CPU

4 vCPU
Minimum: 2 vCPU

RAM

8 GB
Minimum: 4 GB (without Elastic)

Disk

50 GB SSD
Minimum: 20 GB (Prometheus TSDB)

Network

100 Mbps
Stable outbound internet access

Elasticsearch Memory: Elasticsearch requires at least 2 GB dedicated heap. On systems with less than 6 GB of RAM, consider disabling Elasticsearch and Kibana in docker-compose.yaml.

Network Requirements

Hawk-Eye requires outbound connectivity to monitor external targets and inbound access on specific ports for the dashboards. The following firewall rules must be in place:

Port Protocol Direction Service
8080 TCP Inbound Hawk-Eye Dashboard & API
3000 TCP Inbound (internal) Grafana
9090 TCP Inbound (internal) Prometheus
9093 TCP Inbound (internal) Alertmanager
5601 TCP Inbound (internal) Kibana
9200 TCP Inbound (internal) Elasticsearch
80/443 TCP Outbound Monitoring targets (HTTP/HTTPS)
53 UDP Outbound DNS Check probes

For production, expose only port 8080 on the public internet. All other ports (Grafana, Prometheus, etc.) should only be accessible within your private VPC or via VPN.

Pinned Component Versions

The following versions are tested and verified to work together. Mixing versions may cause compatibility issues.

YAMLdocker-compose.yaml — tested versions
Prometheus:       v2.51.0
Grafana:          10.4.0
Alertmanager:     v0.27.0
Elasticsearch:    8.15.0
Kibana:           8.15.0
Thanos:           v0.35.0

Always use matching major versions for Elasticsearch and Kibana (e.g., both 8.15.x). Mismatched versions will prevent Kibana from connecting.
