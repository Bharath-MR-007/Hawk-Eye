Installation Procedure

A complete, step-by-step guide to deploying Hawk-Eye in development, staging, or production environments.

Overview

Hawk-Eye is a distributed infrastructure monitoring suite built on the Hawk-Eye Engine. It supports three deployment methods:

  • Docker Compose (recommended for most teams — spins up the full stack with one command)
  • Binary (lightweight, for embedded or CI/CD use)
  • Helm Chart (production Kubernetes clusters)

For a first-time setup, the Docker Compose method is strongly recommended as it includes Prometheus, Grafana, Alertmanager, Elasticsearch, and Kibana preconfigured.

Prerequisites

Before proceeding, ensure the following tools are installed and accessible from your shell:

BASHCheck installed versions
# Verify Docker
docker --version          # Minimum: Docker 20.10.0

# Verify Docker Compose
docker compose version    # Minimum: v2.0.0

# Verify Go (only needed for building from source)
go version                # Minimum: go1.22

# Verify Git
git --version

Architecture Note: If you are on an Apple Silicon Mac (M1/M2/M3), build with GOARCH=arm64. For Linux servers and Intel Macs, use GOARCH=amd64.

Option 1: Docker Compose (Recommended)

This method launches the full Hawk-Eye observability stack including the monitoring engine, Prometheus, Grafana, Alertmanager, Elasticsearch, and Kibana.

1Clone the Repository

BASH
git clone https://github.com/Bharath-MR-007/hawk-eye.git
cd hawkeye

2Build the Binary for Linux

The Docker image uses a scratch base, so the binary must be pre-compiled for Linux before building the image.

CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build -o hawkeye .
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o hawkeye .

3Configure Monitoring Targets

Edit checks.yaml in the project root to define the endpoints Hawk-Eye should monitor:

YAMLchecks.yaml
health:
  interval: 30s
  timeout: 10s
  retry:
    count: 3
    delay: 2s
  targets:
    - https://google.com
    - https://aws.amazon.com
    - http://your-internal-service:8080/health

latency:
  interval: 60s
  timeout: 10s
  targets:
    - https://api.example.com

dns:
  interval: 60s
  targets:
    - your-domain.internal

4Start the Stack

BASH
docker compose up -d --build

Docker will pull all required images (~2-3 minutes on first run) and start all services in the background.

5Access the Application

Once started, open your browser and navigate to:

URL
http://localhost:8080       → Hawk-Eye Dashboard (admin / admin)
http://localhost:3000       → Grafana
http://localhost:9090       → Prometheus
http://localhost:9093       → Alertmanager
http://localhost:5601       → Kibana

Default login for Hawk-Eye Dashboard: Username: admin / Password: admin

Option 2: Binary Installation (No Docker Required)

Run Hawk-Eye as a standalone process on any Linux server, macOS machine, or VM — no container runtime needed. This is ideal for environments with an existing observability stack (Prometheus, Grafana) or for lightweight CI/CD integration.

This method deploys only the Hawk-Eye monitoring engine. For metrics visualization, point your existing Prometheus and Grafana instances at Hawk-Eye's /metrics endpoint.

1Download the Repository

BASH
git clone https://github.com/Bharath-MR-007/hawk-eye.git
cd hawkeye

2Build the Binary

Build a native binary for your current OS and architecture:

CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o hawkeye .
CGO_ENABLED=0 GOOS=linux GOARCH=arm64 go build -o hawkeye .
CGO_ENABLED=0 go build -o hawkeye .

For ICMP / traceroute checks to work, the binary needs raw socket access. On Linux, either run as root or grant capabilities with: sudo setcap cap_net_raw+ep ./hawkeye

3Create checks.yaml

Define what Hawk-Eye should monitor in checks.yaml:

YAMLchecks.yaml
health:
  interval: 30s
  timeout: 10s
  retry:
    count: 3
    delay: 2s
  targets:
    - https://example.com
    - https://your-internal-service/health

latency:
  interval: 60s
  timeout: 10s
  targets:
    - https://api.example.com

dns:
  interval: 60s
  targets:
    - your-domain.internal

4Run the Binary

BASH
# Basic run
./hawkeye run --hawkeyeName hawkeye.prod --loaderType file --loaderFilePath ./checks.yaml

# With custom API port
./hawkeye run --hawkeyeName hawkeye.prod --apiAddress :9000 --loaderType file --loaderFilePath ./checks.yaml

5Run as a systemd Service (Production Linux)

To make Hawk-Eye start automatically on boot and restart on failure, install it as a systemd unit:

BASHCopy binary to system path
sudo cp hawkeye /usr/local/bin/hawkeye
sudo chmod +x /usr/local/bin/hawkeye
sudo setcap cap_net_raw+ep /usr/local/bin/hawkeye
INI/etc/systemd/system/hawkeye.service 
[Unit]
Description=Hawk-Eye Monitoring Engine
After=network.target

[Service]
Type=simple
User=nobody
ExecStart=/usr/local/bin/hawkeye run \
  --hawkeyeName hawkeye.prod \
  --loaderType file \
  --loaderFilePath /etc/hawkeye/checks.yaml \
  --apiAddress :8080
Restart=always
RestartSec=5s
StandardOutput=journal
StandardError=journal
AmbientCapabilities=CAP_NET_RAW

[Install]
WantedBy=multi-user.target
BASHEnable and start
sudo mkdir -p /etc/hawkeye
sudo cp checks.yaml /etc/hawkeye/checks.yaml

sudo systemctl daemon-reload
sudo systemctl enable hawkeye
sudo systemctl start hawkeye

# Check status
sudo systemctl status hawkeye
sudo journalctl -u hawkeye -f

6Connect to Existing Prometheus & Grafana

Hawk-Eye exposes Prometheus metrics at http://<host>:8080/metrics. Add it as a scrape target in your existing prometheus.yaml:

YAMLprometheus.yaml — add scrape job 
scrape_configs:
  - job_name: 'hawkeye'
    static_configs:
      - targets: ['hawkeye-host:8080']
    scrape_interval: 15s

Then import the Hawk-Eye Grafana dashboards (located in grafana/dashboards/ in the repository) into your existing Grafana instance via Dashboards → Import → Upload JSON.

Once Prometheus is scraping /metrics, all monitoring data (uptime, latency, DNS, SSL) will appear in your Grafana dashboards automatically — no Docker needed!

Option 3: Helm (Production Kubernetes)

1Add the Helm repository

BASH
helm repo add hawkeye oci://ghcr.io/Bharath-MR-007/charts/hawk-eye
helm repo update

2Create a values override file

YAMLvalues-prod.yaml
startupConfig:
  name: hawkeye.mycompany.com
  loader:
    type: file
    interval: 30s

checksConfig:
  health:
    interval: 30s
    targets:
      - https://api.example.com/health

3Install into Kubernetes

BASH
kubectl create namespace monitoring
helm upgrade -i hawkeye oci://ghcr.io/Bharath-MR-007/charts/hawk-eye \
  --namespace monitoring \
  -f values-prod.yaml

Verify Installation

After starting, run these commands to confirm all services are healthy:

BASH
# Check running containers
docker compose ps

# Confirm hawkeye API is responding
curl -s http://localhost:8080/metrics | head -20

# Check logs for errors
docker compose logs hawkeye --tail=50

Troubleshooting

Container exits immediately

BASH
docker compose logs hawkeye
# Look for: "failed to parse config" or "permission denied"

Ensure checks.yaml is valid YAML and the binary was built for the correct architecture (linux/arm64 vs linux/amd64).

Kibana fails to start (manifest unknown)

BASH
docker compose pull kibana
docker compose up -d kibana

Ensure you are using a valid image version. Check elastic.co for the latest available tags.

For more assistance, check the server logs at hawkeye.log in the project root during local development.
