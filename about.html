Introduction

Hawk-Eye is a full-stack network observability and infrastructure monitoring platform built on the Hawk-Eye Engine. It provides SRE and platform engineering teams with real-time visibility into the health, latency, DNS resolution, TCP connectivity, SSL/TLS validity, and network path of distributed infrastructure components across all layers of the stack.

Unlike traditional monitoring tools that only track resource utilization (CPU, RAM), Hawk-Eye focuses on black-box probing — testing your services from the outside, exactly as your users or dependent systems would experience them. This gives teams an objective, user-perspective measure of service health.

Mission

To give platform teams the eyes of a hawk — the ability to detect network degradation, endpoint failures, and latency spikes before your customers do. Hawk-Eye achieves this through:

  • Proactive detection: Continuous check loops with sub-30-second detection times
  • Contextual alerting: Alerts enriched with target metadata, check history, and severity levels
  • Operational clarity: A purpose-built dashboard that gives operators one clear, live view without needing to understand Prometheus query language
  • Portability: Full configuration export/import to enable DR failover onto a fresh instance in minutes

Origin & Project History

Hawk-Eye is a production deployment layer built on top of the Hawk-Eye project, originally developed and open-sourced by Deutsche Telekom's internal platform teams. Hawk-Eye was created to solve the challenge of monitoring internal API health and network latency between T-Systems data centers distributed across Europe.

The Hawk-Eye layer adds:

  • A production-ready Docker Compose deployment stack
  • A purpose-built live dashboard with WebSocket updates
  • Integrated Alertmanager, Grafana, Elasticsearch, and Kibana
  • Target grouping, tagging, and a one-click export/import system

Technology Stack

  • Backend: Go 1.22+ — high-performance concurrent probing goroutines
  • Web Framework: go-chi/chi — lightweight HTTP router for the REST API
  • Frontend: Vanilla HTML/CSS/JavaScript — WebSocket-driven live updates, no SPA framework required
  • Time-Series DB: Prometheus — metric collection, alerting rule evaluation
  • Long-term Storage: Thanos — multi-cluster federation and block storage
  • Log Storage: Elasticsearch 8.x — structured log and trace storage
  • Observability: OpenTelemetry — distributed tracing from the Go engine
  • Visualization: Grafana 10.x + Kibana 8.x
  • Alerting: Prometheus Alertmanager — multi-channel notification routing
  • Container Runtime: Docker / scratch-based image for minimal attack surface
