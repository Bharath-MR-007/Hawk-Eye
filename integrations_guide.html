Integration Guide

This guide explains how to connect Hawk-Eye with your enterprise monitoring stack. Whether you are a newcomer setting up for the first time or an administrator configuring a distributed multi-server environment, follow the steps below for each tool.

Grafana Prometheus Alertmanager Elastic Stack Kibana SNMP Integration Remote Server

What is Hawk-Eye?

Hawk-Eye is an enterprise network and application monitoring platform built by T-Systems. It performs health checks (HTTP, TCP, DNS, SSL, Traceroute) at customizable intervals, exposes a Prometheus-compatible /metrics endpoint, and surfaces incidents in a real-time operational dashboard.

External tools like Grafana and Kibana are not required for Hawk-Eye to operate — but they supercharge visibility, alerting, and log analysis across your organization.

Integration Architecture

Hawk-Eye sits at the center of your observability stack. Here is how data flows between components:

Architecture Overview
  ┌─────────────────────────────────────────────────────┐
  │                   Hawk-Eye (port 8080)              │
  │   Checks: HTTP · TCP · DNS · SSL · Traceroute      │
  │   Exposes: /metrics  /v1/metrics/{check}           │
  └──────┬──────────────────────────┬──────────────────┘
         │  scrape /metrics         │  push alerts
         ▼                          ▼
  ┌──────────────┐          ┌──────────────────┐
  │  Prometheus  │─────────▶│  Alertmanager    │
  │  (port 9090) │ alerts   │  (port 9093)     │
  └──────┬───────┘          └──────┬───────────┘
         │ query                   │ notify
         ▼                         ▼
  ┌──────────────┐         Slack / PagerDuty / Email
  │   Grafana    │
  │  (port 3000) │
  └──────────────┘

  Hawk-Eye Logs ──► Filebeat ──► Elasticsearch (9200) ──► Kibana (5601)
All services can run on the same host via Docker Compose (included) or on separate servers. The Integration Config page lets you define the URLs for each external service.

Grafana

Grafana visualizes Hawk-Eye metrics scraped by Prometheus. Use it to build dashboards for latency trends, success rates, and SLA reporting.

Quick Start

1

Grafana is pre-configured in Docker Compose

If you're running the included docker-compose.yaml, Grafana starts automatically on port 3000. Login with admin / admin.

2

Add Prometheus as a Data Source

Go to Connections → Data Sources → Add data source → Prometheus. Set the URL to where Prometheus is running.

3

Import or Build a Dashboard

Navigate to Dashboards → Import and upload the JSON files from the /grafana/dashboards/ directory in this project.

Configuring the Prometheus Data Source

If Grafana and Prometheus are on the same host:

Grafana Datasource URL (same host)
URL: http://localhost:9090

If Prometheus runs on a remote server (e.g. IP 192.168.1.50):

Grafana Datasource URL (remote server)
URL: http://192.168.1.50:9090

If Grafana is inside Docker and Prometheus is also in the same Docker Compose network:

Grafana Datasource URL (Docker internal)
URL: http://prometheus:9090

Prometheus

Prometheus scrapes the Hawk-Eye /metrics endpoint at regular intervals and stores time-series data used by Grafana and Alertmanager.

Scrape Configuration

Add the following job to your prometheus.yaml:

prometheus.yaml — scrape config
scrape_configs:
  - job_name: 'hawk-eye'
    scrape_interval: 15s
    scrape_timeout: 10s
    static_configs:
      - targets:
          - 'hawkeye:8080'      # Docker internal hostname
          # - '192.168.1.100:8080'  # Remote server IP
    metrics_path: '/metrics'
Verify Prometheus is scraping successfully by visiting http://localhost:9090/targets. The Hawk-Eye target should show State: UP.

Hawk-Eye also exposes per-check metrics at:

Per-check metric endpoints
GET /v1/metrics/health
GET /v1/metrics/latency
GET /v1/metrics/dns
GET /v1/metrics/ssl
GET /v1/metrics/traceroute

Alertmanager

Alertmanager receives firing alerts from Prometheus and routes them to notification channels such as Slack, email, or PagerDuty.

Routing Configuration

alertmanager.yaml — basic routing
global:
  resolve_timeout: 5m

route:
  group_by: ['alertname', 'target']
  group_wait: 30s
  group_interval: 5m
  repeat_interval: 1h
  receiver: 'slack-notifications'

receivers:
  - name: 'slack-notifications'
    slack_configs:
      - api_url: 'https://hooks.slack.com/services/YOUR/WEBHOOK/URL'
        channel: '#alerts'
        text: '{{ .CommonAnnotations.summary }}'

Elastic Stack (ELK)

Hawk-Eye emits structured JSON logs. These can be shipped to Elasticsearch via Filebeat for deep search, audit trails, and anomaly detection.

filebeat.yaml — send Hawk-Eye logs to Elasticsearch
filebeat.inputs:
  - type: log
    paths:
      - /path/to/hawkeye.log
    json.keys_under_root: true
    json.add_error_key: true

output.elasticsearch:
  hosts: ["http://elasticsearch:9200"]
  index: "hawk-eye-logs-%{+yyyy.MM.dd}"

Kibana — Viewing Logs

1

Open Kibana

Navigate to http://localhost:5601. Kibana is pre-started in the Docker Compose stack.

2

Create Index Pattern

Go to Stack Management → Index Patterns and create pattern hawk-eye-logs-*.

3

Explore in Discover

Navigate to Discover, select your index pattern, and filter by fields like level, target, or check.

SNMP Trap Integration

Hawk-Eye can send SNMP v2c traps to your enterprise Network Management System (NMS) whenever an incident is detected or resolved. This is ideal for integration with tools like HP OpenView, Zabbix, or IBM Netcool.

snmp.yaml — standalone trap configuration
snmp_config:
  target: "nms-server.internal"
  port: 162
  community: "public"
  version: "v2c"
  trap_oid: "1.3.6.1.4.1.2021.251.1"

Enterprise Integration

Traps include the following variable bindings (VarBinds):

  • AlertName: The name of the firing alert.
  • Severity: critical, warning, or info.
  • Instance: The target being monitored.
  • Description: Human-readable error message.

Remote Server Configuration

When any of these tools run on a different server, update the URLs in the Integration Config page and ensure the correct ports are open in your firewall.

Always use internal/private network IPs for inter-service communication. Never expose unauthenticated Prometheus or Elasticsearch ports on a public interface.

Example: Hawk-Eye running on 10.0.0.10:8080, Prometheus on 10.0.0.11:9090, Grafana on 10.0.0.12:3000:

Remote multi-server setup — prometheus.yaml
scrape_configs:
  - job_name: 'hawk-eye-remote'
    static_configs:
      - targets: ['10.0.0.10:8080']

Troubleshooting

Prometheus target shows DOWN

Check that Hawk-Eye is running and the URL/port is correct. Inside Docker, use the service name (hawkeye), not localhost.

Grafana shows "No data"

Verify the data source URL, then check that the time range in the Grafana panel matches when Prometheus started scraping. Try running a raw PromQL query like up{job="hawk-eye"} in Prometheus.

Alerts not being sent

Check http://localhost:9093 to see if Alertmanager received the alert. Verify the webhook URL and that the prometheus_rules.yaml is loaded correctly.

Kibana shows no index

Ensure Filebeat is running and that logs are being written to the path specified in filebeat.yaml. Check Elasticsearch health at http://localhost:9200/_cluster/health.

For advanced configuration, connection parameters for all integrations can be managed from the External Apps Integration Config page. Changes are saved automatically and affect the quick-links on the External Apps Links page.
